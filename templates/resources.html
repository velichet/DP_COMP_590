Differential Privacy

Differential Privacy (DP) was introduced by Dwork et al. This seminal work has paved the way for Differential Privacy as numerous companies and governments have adopted this technology.

What is DP?

DP allows for datasets containing sensitive data to be shared without exposing individual data points. This is done by injecting noise (randomness) into the dataset. This video from NIST explains DP.

How does DP work?

DP requires data curaors (third party) to collect data from data producers. The curator allows data consumers to query the datasets. The curator answers these queries with differentially private responses.

What is Local DP?

Local DP is a model of DP in which the data that is sent to the curator (third party) is injuected with noise before the curator has access to the data. This makes it possible for the third party to be untrusted.

What is Global DP?

Global (or central) DP is a model of DP in which raw data is sent to the curator (third party). This requires a trusted-third party to be available, however, the query results tend to be more accurate in this model.

Choosing Epsilon

In DP, epsilon is the privacy parameter. The closer to zero that epsilon is, the more randomness will be injected into the dataset. Checkout datasets by Safely Stat in The Gallery to see the effect of epsilon.

Using Safely Stat

Navigating

Safely Stat currently consists of a few pages: Home, Resources, Gallery, MyData. Home and Resources provide information on DP and other important information on using the platform. The Gallery contains all uploaded datasets. MyData contains profile information, user's uploaded datasets, and serves as the portal to uploaded more datasets.

Upload Datasets

  1. Go to MyData

  2. For Global DP press Global Uploads. For Local DP press Local Uploads.

  3. Fill out all of the information on the form. Verify that all the information is correct.

  4. Upload your dataset in CSV format. Once you do this your Gallery Piece will be generated.

Proper Formatting

Ensure your CSV is formatted properly. The below exemplifies proper formatting. age,height,weight 30,70,200 53,65,130 44,58,105

Algorithms

Safely Stat

This project would not be possible without the work of OpenMinded and Google. The open source Differential Privacy projects of these two organizations has made this site possible. To see how we implemented these algorithms, visit the Safely Stat repo.

OpenMinded

Safely Stat's Global DP solution makes use of OpenMinded's Python Laplacian functions. To learn more about this project, click on the GitHub icon below.

Google

Safely Stat's Local DP solution is made in-house, however, heavy inspiration was drawn from Google's RAPPOR solution. Click on the icon to learn more.
